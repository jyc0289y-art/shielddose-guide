ID Interconnection System -- Design Principles (Knowledge Graph / Linked Data) ▶

1. Knowledge Graph (Google, 2012)

Entity-Relationship graph representation. Each ID (PIT-025, LT-04, ISS-020) is a node; cross-references are edges with typed relations (resolved-by, physics-root, addressed-by). Enables traversal: "PIT-025 is addressed by LT-04, which resolves ISS-020."

2. Linked Data (Tim Berners-Lee, 2006) -- 4 Principles

(1) URI identification: PIT-025, LT-04 act as URIs
(2) HTTP URI access: anchor links #PIT-025 enable direct navigation
(3) Standard format: HTML modal provides structured info on dereference
(4) Links to other URIs: bidirectional references (forward + back)

3. Zettelkasten (Niklas Luhmann)

Each note (ISS/PIT/LT item) has a unique ID and cross-references other notes. The emergent knowledge structure arises from the web of connections, not from any single note. Back-references reveal unexpected clusters.

4. Traceability Matrix (CMMI / ISO 26262)

Bidirectional traceability between Requirements (REQ) -> Design (LT) -> Implementation (ISS) -> Verification (PIT). Every item can be traced forward to its implementation and backward to its origin requirement.

This v2 dashboard implements all four principles: every ID is an anchor node, every badge is a navigable link, the detail panel shows forward + back references (computed from the knowledge graph), and the cluster view provides traceability from REQ through LT/PIT to ISS.
