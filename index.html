GEANT4 C++

LAYER 1 — CERN이 만든 것
GEANT4 Library

02용어 사전

ShieldDose 프로젝트에서 사용되는 주요 용어를 6개 카테고리로 정리합니다. 총 70+ 용어를 수록.

Physics (방사선 물리)

용어 뜻 우리 프로젝트에서
HVL
Half-Value Layer		 방사선의 선량률을 50%로 줄이는 데 필요한 차폐 두께.
물질과 에너지에 따라 값이 다름. 차폐 설계의 기본 단위. 		MC 시뮬레이션으로 6개 핵종 전부 HVL 결정 완료 (P2.6wc).
38회 두께 스캔으로 독립 측정. QSA 매뉴얼 기준 PASS.
TVL
Tenth-Value Layer		 방사선의 선량률을 10%로 줄이는 데 필요한 차폐 두께.
TVL ≈ 3.32 × HVL (빌드업 무시 시). 		HVL과 함께 차폐 설계 도구에서 참고값으로 표시.
감쇠계수
μ, Linear Attenuation Coefficient		 물질이 방사선을 흡수하는 정도를 나타내는 단위 길이당 감쇠율 [cm¹].
I = I⋅exp(-μx) — Beer-Lambert 법칙의 핵심 파라미터. 		NIST XCOM 데이터베이스에서 에너지별 μ 값을 사용.
해석적 계산에서 exp(-μx) 항으로 차폐 효과를 계산.
빌드업 팩터
Buildup Factor, B		 산란선 기여를 보정하는 계수.
좁은 빔(narrow beam) 감쇠 대비 넓은 빔(broad beam)에서의 실제 선량 증가분.
B ≥ 1 (항상). 두꺼운 차폐일수록 B가 크게 증가. 		ANS 6.4.3 표준 데이터로 에너지-MFP 이중선형 보간 (P2.12wc).
다층 차폐 시 Kalos-Kitazume 방법 적용.
Compton 산란
Compton Scattering		 감마선이 전자에 의해 산란되며 에너지가 감소하는 과정.
중간 에너지 영역(수백 keV~수 MeV)에서 지배적인 상호작용. 		GEANT4 option4 물리 리스트에서 정밀 모델링.
빌드업 팩터가 커지는 주 원인.
광전 효과
Photoelectric Effect		 감마선이 원자에 완전 흡수되어 전자가 방출되는 과정.
저에너지 영역에서 지배적. Z가 높을수록 확률 증가 (∝ Z&sup5;). 		납(Z=82)이 저에너지 감마선 차폐에 효과적인 이유.
Tc-99m(140 keV)은 광전 효과가 주요 감쇠 메커니즘.
쌍생성
Pair Production		 1.022 MeV 이상의 감마선이 원자핵 근처에서 전자-양전자 쌍을 생성하는 과정.
고에너지 영역에서 지배적. 		Co-60(1.17+1.33 MeV)에서 일부 기여.
GEANT4 option4에서 자동 처리.
등방 방출
Isotropic Emission		 모든 방향으로 균일하게 방사선을 방출하는 것.
실제 밀봉선원의 방출 패턴에 해당. 		GPS 설정: /gps/ang/type iso
4π 등방 방출로 실제 선원을 모사.
역제곱 법칙
Inverse Square Law		 거리 제곱에 반비례하여 선량률이 감소하는 법칙.
DoseRate ∝ 1/d² — 거리를 2배로 늘리면 선량률은 1/4. 		해석적 계산의 Γ × A / d² 항에 반영.
MC 시뮬레이션에서는 자동으로 구현됨 (기하학적 결과).

Dosimetry (선량 측정)

용어 뜻 우리 프로젝트에서
흡수선량
Absorbed Dose, Gy		 물질 단위질량당 흡수된 에너지.
단위: Gy (Gray) = J/kg. 방사선의 물리적 효과를 나타내는 기본 선량. 		MC 시뮬레이션의 직접 출력 (MeanDose_Gy).
G4PSDoseDeposit 스코어러가 Gy 단위로 기록.
선량률
Dose Rate		 단위시간당 선량 (mSv/h, μSv/h).
방사선 방호에서 가장 중요한 실용 지표. 		DoseRate = MeanDose × Activity × 3600
결과 보고서의 핵심 출력값.
감마상수
Γ, Gamma Constant		 핵종 고유의 선량률 환산 계수.
단위 방사능(1 MBq)이 1m 거리에서 발생시키는 선량률. 		6핵종 전부 ±10% 이내 검증 PASS (P2.3wc~P2.5wc).
QSA Global 880 Series 매뉴얼 기준값 사용.
H*(10)
Ambient Dose Equivalent		 주변선량당량. ICRP-74 기반 방사선 방호 선량.
조직 등가 구체 10mm 깊이에서의 선량당량. 		보고서에서 규제 기준 비교에 사용.
감마선의 경우 흡수선량 ≈ H*(10) (선질계수 ≈ 1).
공기커마율
Air Kerma Rate		 공기 중 방사선의 에너지 전달률.
Kerma = Kinetic Energy Released per unit MAss. 		감마상수 정의에서 사용되는 기본 물리량.
Γ = Air Kerma Rate per unit activity at 1m.
에너지 침적
Energy Deposit		 입자가 물질에 전달한 에너지 [MeV].
흡수선량 계산의 원시 데이터. 		G4PSEnergyDeposit 스코어러로 기록.
Dose = ΣEdep / (밀도 × 부피).

Nuclear (핵물리)

용어 뜻 우리 프로젝트에서
핵종
Nuclide		 특정 원자번호(Z) + 질량수(A) 조합의 원자핵.
예: Co-60 (Z=27, A=60), Cs-137 (Z=55, A=137). 		6개 핵종 지원: Co-60, Cs-137, Ir-192, Se-75, I-131, Tc-99m.
동위원소 선택 기능으로 확장 가능 (P2.9wc).
붕괴도식
Decay Scheme		 핵종의 붕괴 경로와 방출 방사선 정보.
에너지, 분기비, 감마선/베타선 등을 도식으로 표현. 		GPS 매크로 파일이 붕괴도식 기반으로 작성됨.
Co-60: 2개 감마선 캐스케이드 (1.173 + 1.332 MeV).
방사능
Activity, Bq/Ci		 단위시간당 붕괴 수.
1 Bq = 1 붕괴/초, 1 Ci = 3.7×1010 Bq. 		GUI에서 MBq, GBq, Ci 단위로 입력.
선량률 계산: DoseRate ∝ Activity.
분기비
Branching Ratio		 특정 붕괴 경로의 확률.
예: Cs-137에서 662 keV 감마선 방출 확률 = 85.1%. 		GPS User Histogram에서 분기비를 에너지 가중치로 반영.
Ir-192(14개 선), Se-75(8개 선) 등 다중선 핵종.

GEANT4 (시뮬레이션)

용어 뜻 우리 프로젝트에서
Monte Carlo
MC 시뮬레이션		 난수(랜덤)를 사용해 물리 과정을 통계적으로 모사하는 방법.
입자 하나하나의 궤적을 추적하여, 수만~수백만 개의 입자를 시뮬레이션합니다.
이벤트 수가 많을수록 결과가 정밀해집니다. 		GEANT4 = Monte Carlo 시뮬레이션 엔진.
ShieldDose에서 보통 50만~100만 이벤트를 실행합니다.
G4RunManager GEANT4 시뮬레이션의 총괄 관리자.
기하 초기화, 물리 초기화, 이벤트 루프 실행, 결과 수집을 담당. 		ShieldDose CLI의 main.cc에서 생성.
/run/beamOn 500000 명령으로 이벤트 실행.
MT Mode
Multi-Threading		 여러 CPU 코어로 병렬 시뮬레이션.
G4MTRunManager를 사용하여 이벤트를 코어별로 분배. 		현재 ShieldDose는 Serial Mode(단일 코어).
LT-04로 MT Mode 전환 예정 (Phase 1 완료, P2.10wc).
Serial Mode 단일 코어 순차 실행.
G4RunManager 사용. 디버깅이 쉽고 결정론적 재현이 가능. 		현재 ShieldDose CLI의 실행 방식.
검증 단계에서 재현성을 위해 유지.
G4GeneralParticleSource
GPS		 GEANT4의 입자 소스 정의 시스템.
입자 종류, 에너지, 각도 분포를 매크로로 설정. 		6개 핵종별 GPS 매크로 작성 완료.
Mono, User Histogram, Multiple Vertex 방식 사용.
SteppingAction 입자 이동의 각 단계(step)마다 실행되는 콜백.
에너지 침적, 경계 통과 등을 기록하는 데 사용. 		디텍터 영역에서의 에너지 침적을 기록.
선량 스코어링의 핵심 컴포넌트.
MFD
MultiFunctionalDetector		 복수 물리량을 동시에 측정하는 디텍터 구조.
하나의 디텍터에 여러 스코어러(선량, 에너지, 플럭스)를 연결. 		ShieldDose에서 DoseDeposit + EnergyDeposit 동시 측정.
Importance Biasing 중요도 기반 분산감소 기법.
관심 영역(디텍터)에 입자가 도달할 확률을 인위적으로 높임. 		차폐벽을 N층으로 분할, 깊은 층에서 입자를 분열시켜 통계 확보.
상대오차 99.9% → <5% 달성 (P2.3wc).
Variance Reduction
VR, 분산감소		 통계 효율 향상을 위한 기법 총칭.
Importance Biasing, Russian Roulette, Weight Window 등 포함. 		ShieldDose에서는 Importance Biasing을 자동 최적화.
사용자가 수동 설정할 필요 없음.

Software (소프트웨어)

용어 뜻 우리 프로젝트에서
CLI
Command Line Interface		 텍스트 명령어로 조작하는 프로그램.
마우스로 클릭하는 대신 터미널에 명령어를 입력해서 실행합니다.
예: ./ShieldDose run.mac 		ShieldDose 바이너리.
GEANT4 엔진을 내장하고, .mac 파일을 읽어 시뮬레이션 실행 후 CSV 출력.
GUI
Graphical User Interface		 마우스/터치로 조작하는 시각적 프로그램.
버튼, 드롭다운, 진행률 바 등 시각 요소로 구성됩니다. 		ShieldDoseGUI.
사용자가 핵종/차폐를 선택하면, 내부적으로 CLI를 호출하여 시뮬레이션 실행.
바이너리
Binary / Executable		 컴파일된 실행 파일. 소스코드(C++)를 기계어로 변환한 결과물.
더블클릭 또는 터미널에서 직접 실행 가능. 		ShieldDose (CLI 바이너리), ShieldDoseGUI (GUI 바이너리)
라이브러리
Library		 다른 프로그램이 가져다 쓸 수 있는 코드 모음.
단독 실행 불가 — 반드시 다른 프로그램에 포함(링크)되어 사용됩니다. 		GEANT4 = C++ 라이브러리. ShieldDose CLI가 이 라이브러리를 링크하여 사용.
링크
Linking		 컴파일 시 라이브러리의 기능을 프로그램에 결합하는 과정.
"GEANT4에 링크한다" = GEANT4 라이브러리의 코드를 프로그램에 포함시킨다. 		ShieldDose CLI는 GEANT4에 링크됨 (GEANT4 코드가 내장됨).
GUI는 GEANT4에 링크하지 않음 (Qt5만 사용).
매크로 파일
.mac file		 GEANT4에게 "무엇을 시뮬레이션할지" 알려주는 텍스트 명령어 파일.
입자 종류, 에너지, 기하 설정, 이벤트 수 등을 지정합니다. 		macros/sources/co60.mac 등. GUI가 이 파일을 자동 생성.
Qt5 C++로 GUI 프로그램을 만들 수 있게 해주는 프레임워크.
버튼, 텍스트 입력, 테이블, 프로그레스 바 등 UI 부품을 제공합니다.
물리 시뮬레이션과는 관계 없음 — 순수하게 화면 표시 담당. 		ShieldDose GUI의 모든 시각적 요소를 Qt5로 구현.
CMake C++ 프로젝트의 빌드 설정 도구.
"어떤 소스 파일을 어떤 라이브러리와 결합해서 어떤 실행 파일을 만들지"를 정의합니다.
CMakeLists.txt 파일에 설정을 작성합니다. 		CLI용 CMakeLists.txt: GEANT4 + 소스코드 → ShieldDose 바이너리
GUI용 CMakeLists.txt: Qt5 + GUI 소스코드 → ShieldDoseGUI 바이너리
Makefile CMake의 구식 대안. make 명령어로 빌드를 실행합니다.
CMake가 더 현대적이고 크로스 플랫폼을 지원하여 신규 프로젝트에서 선호됩니다. 		기존 GUI (Phase 2)에서 사용했으나, Phase 3에서는 CMake로 전환.
프로세스
Process		 실행 중인 프로그램. OS가 메모리를 할당하고 관리하는 단위.
하나의 프로그램에서 다른 프로그램을 "프로세스 생성(spawn)"하여 실행할 수 있습니다. 		GUI 프로세스가 CLI 프로세스를 생성(spawn)하여 시뮬레이션 실행.
두 프로세스는 독립적 — GUI가 죽어도 시뮬레이션은 계속됩니다.
CSV Comma-Separated Values. 쉼표로 구분된 텍스트 데이터 파일.
엑셀에서 열 수 있는 간단한 표 형식입니다. 		GEANT4가 시뮬레이션 결과를 CSV로 출력.
GUI가 이 CSV를 읽어서 시각화합니다.
STL 파일 3D 형상을 삼각형 메시로 표현한 파일 포맷.
CAD 프로그램(Onshape, SolidWorks 등)에서 내보내기 가능. 		Onshape에서 차폐체 형상을 STL로 내보내 → ShieldDose에 입력.

Standards (표준/규제)

03GEANT4란 무엇인가

GEANT4 = GEometry ANd Tracking, version 4

CERN(유럽입자물리연구소)이 개발한 C++ 입자 물리 시뮬레이션 프레임워크.
1998년 첫 배포 이래, 전 세계 수천 개 연구기관에서 사용 중.
고에너지 물리, 의학물리, 우주방사선, 방사선 방호 등에 활용.

GEANT4가 하는 일

GEANT4는 입자 하나하나의 여행(궤적)을 물리 법칙에 따라 추적합니다:

감마선 광자 발생 (선원에서)
    │
    ▼ 직진 이동 (랜덤 거리)
    │
    ├─ Compton 산란 → 방향 변경 + 에너지 감소 → 계속 이동
    │
    ├─ 광전 효과  → 광자 흡수 → 전자 방출 → 추적 종료
    │
    ├─ 쌍생성     → e⁺ + e⁻ 생성 (>1.022 MeV)
    │
    └─ 디텍터 도달 → 에너지 침적(deposit) 기록 → 선량 계산

이 과정을 50만 번 반복하면 통계적으로 의미 있는 결과를 얻습니다. 이것이 Monte Carlo 방법입니다.

GEANT4가 제공하는 것

물리 모델

전자기 상호작용, 핵반응, 붕괴 등 수백 가지 물리 과정의 수학적 모델. CERN이 수십 년간 실험으로 검증.

입자 추적 엔진

3D 공간에서 입자의 이동, 상호작용, 에너지 손실을 단계별로 계산하는 핵심 엔진.

기하 시스템

상자, 원통, 구 등 기본 형상과 STL 메시 등을 조합하여 시뮬레이션 공간을 구성.

물질 데이터베이스

G4_Pb(납), G4_WATER(물), G4_CONCRETE(콘크리트) 등 수백 종의 물질과 그 물리적 특성.

중요: GEANT4는 라이브러리입니다. 단독으로 실행할 수 없습니다.
반드시 사용자가 C++ 프로그램을 작성하여 GEANT4 라이브러리를 호출해야 합니다.
이 "사용자가 작성하는 C++ 프로그램"이 바로 ShieldDose CLI입니다.

Step 1: C++ 코드 작성 (연구자가 직접)

   │  "Co-60 감마선을 등방(모든 방향)으로 쏴라"

G4EmStandardPhysics_option4 — 물리 리스트 (물리 법칙 선택) 
물리 리스트 = "어떤 물리 법칙을 켤 것인가"

  GEANT4는 수백 가지 물리 과정을 제공합니다.
  모두 켜면 너무 느려지고, 잘못 고르면 결과가 틀립니다.

  option4 = 감마선/전자 시뮬레이션에 최적화된 설정
  ┌──────────────────────────────────────┐
  │  켜져 있는 물리 과정:                  │
  │  ✓ Compton 산란     (감마 → 감마 + e⁻)│
  │  ✓ 광전 효과        (감마 → e⁻)       │
  │  ✓ 쌍생성           (감마 → e⁺ + e⁻)  │
  │  ✓ Rayleigh 산란    (감마 → 감마)     │
  │  ✓ 전자 이온화      (e⁻ → e⁻ + e⁻)   │
  │  ✓ 제동복사         (e⁻ → e⁻ + 감마)  │
  │  ✓ 방사성 붕괴                        │
  └──────────────────────────────────────┘

  왜 option4?
  option1~3보다 낮은 에너지에서 더 정확함.
  방사선 방호에서는 수 keV~수 MeV 범위가 중요하므로
  option4가 최적의 선택입니다.
G4GeneralParticleSource (GPS) — 입자 소스 (방사선 선원) 
GPS = "어떤 방사선을 어디서 어떻게 쏘는가"

  3가지 설정 요소:

  1. 입자 종류    /gps/particle gamma
     감마선, 전자, 중성자, 양성자 등

  2. 에너지 스펙트럼
     ┌── Mono (단일 에너지)
     │   /gps/ene/type Mono
     │   /gps/ene/mono 0.662 MeV        ← Cs-137
     │
     ├── User Histogram (다중 에너지)
     │   /gps/ene/type User
     │   /gps/hist/point 0.2955 0
     │   /gps/hist/point 0.2965 28.7    ← 296 keV (28.7%)
     │   /gps/hist/point 0.3075 0       ← Ir-192 등
     │
     └── Multiple Vertex (캐스케이드)
         /gps/source/multiplevertex true ← Co-60
         1.173 MeV + 1.332 MeV 동시 방출

  3. 각도 분포    /gps/ang/type iso
     iso = 등방 (모든 방향 균등)

  G4PSFlatSurfaceFlux — 표면 플럭스 (참고용)
     디텍터 표면을 통과하는 감마선 수

  선량 → 선량률 변환:
  DoseRate [Gy/h] = MeanDose [Gy/event] × Activity [Bq] × 3600 [s/h]

  예: MeanDose = 1×10⁻¹⁶ Gy, Activity = 37 GBq (1 Ci)
     → DoseRate = 1×10⁻¹⁶ × 3.7×10¹⁰ × 3600 = 13.3 mGy/h
G4ImportanceBiasing — 분산감소 (효율 향상 기법) 
문제: 두꺼운 차폐벽을 통과하는 감마선은 극소수

  50만 이벤트 중 차폐벽 뒤 디텍터에 도달하는 감마선:
  차폐 없음  → ~6,500개 도달 (1.3%)
  5cm Pb    → ~10개 도달 (0.002%)  ← 통계적으로 무의미!

해결: Importance Biasing (중요도 편향)

  차폐벽을 여러 층으로 나누고, 깊은 층에 들어갈수록
  입자를 "분열"시켜 통계를 인위적으로 늘림:

  ┌─────┬─────┬─────┬─────┬─────┐
  │ Pb  │ Pb  │ Pb  │ Pb  │ Pb  │  ← 차폐벽을 5층으로 분할
  │ ×1  │ ×2  │ ×4  │ ×8  │ ×16 │  ← 각 층의 "중요도"
  └─────┴─────┴─────┴─────┴─────┘

  입자가 ×2 → ×4 경계를 넘으면 2개로 복제 (가중치 반감)
  → 깊은 곳에서도 충분한 통계를 확보
  → 가중치 보정으로 물리적 정확도 유지

  결과:
  상대오차 99.9995% → <5% (P2.3wc에서 구현)

09해석적 선량 계산

MC 시뮬레이션과 별도로, GUI 내부에서 해석적(Analytical) 계산을 수행하여 결과를 교차 검증합니다.

기본 공식

차폐 후 선량률 계산 공식

  DoseRate = Γ × A / d² × B(μx) × exp(-μx)

  각 항의 의미:
  ┌──────────────────────────────────────────────────────┐
  │  Γ (감마상수)    핵종 고유의 선량률 환산 계수            │
  │                  단위: μSv·m²/(GBq·h)                │
  │                                                      │
  │  A (방사능)      선원의 방사능 [GBq]                    │
  │                                                      │
  │  d² (거리 제곱)   선원-측정점 거리 [m]                  │
  │                  → 역제곱 법칙 적용                    │
  │                                                      │
  │  B(μx) (빌드업)   산란선 보정 계수 (B ≥ 1)              │
  │                  차폐가 두꺼울수록 B가 크게 증가        │
  │                                                      │
  │  exp(-μx) (감쇠)  차폐 물질에 의한 지수 감쇠             │
  │                  μ = 선 감쇠계수, x = 차폐 두께        │
  └──────────────────────────────────────────────────────┘

  예시: Co-60, 1 Ci (37 GBq), 납 5 cm, 거리 30 cm

  Γ = 1.30 μSv·m²/(GBq·h)
  A = 37 GBq
  d = 0.30 m
  μ = 0.638 cm⁻¹ (Pb, ~1.25 MeV)
  x = 5 cm
  B(μx=3.19) ≈ 2.42

  DoseRate = 1.30 × 37 / 0.30² × 2.42 × exp(-3.19)
           ≈ 29.8 mSv/h

에너지 의존 빌드업 팩터 (ANS 6.4.3)

P2.12wc에서 구현된 LT-09 과제로, ANS 6.4.3 표준 데이터를 사용하여 에너지와 차폐 두께(MFP)에 따른 정밀 빌드업 팩터를 계산합니다.

빌드업 팩터 보간 방법

  ANS 6.4.3 테이블:
  ┌──────────┬──────────────────────────────────┐
  │ Energy   │   MFP (Mean Free Path, μx)       │
  │ [MeV]    │  1    2    4    7    10   15   20 │
  ├──────────┼──────────────────────────────────┤
  │ 0.5      │ 2.45 4.19 9.94 ...              │
  │ 1.0      │ 2.13 3.53 7.62 ...              │
  │ 1.5      │ 1.95 3.14 6.46 ...              │
  │ ...      │                                  │
  └──────────┴──────────────────────────────────┘

  보간 방법: log(E) × log(MFP) 이중선형 보간

  1. 주어진 E에서 인접한 두 에너지 행 선택
  2. 주어진 MFP에서 인접한 두 MFP 열 선택
  3. 4개 꼭짓점에서 이중선형 보간 수행
  4. log 스케일에서 보간 → 선형보다 정밀

  다중 에너지 핵종 (Ir-192, Se-75 등):
  각 감마선 에너지별로 B를 개별 계산 후
  분기비(branching ratio)로 가중 평균

다층 차폐의 빌드업: Kalos-Kitazume 방법

Kalos-Kitazume 방법은 N-layer 다층 차폐에서의 빌드업 팩터를 근사하는 방법입니다.

원리: 각 층의 빌드업을 등가 MFP 기준으로 결합
B_total = B₁(μ₁x₁) × B₂((μ₁x₁ + μ₂x₂) / μ₂) / B₂(μ₁x₁ / μ₂) × ...

장점: 단일 물질 빌드업 데이터만으로 다층 계산 가능
한계: 층 순서에 따라 오차 발생 가능 (고Z → 저Z 순서가 정확도 높음)

MC vs 해석적 비교

Monte Carlo (GEANT4)

  • 입자 하나하나 추적 (first principles)
  • 3D 기하, 산란, 2차 입자 모두 포함
  • 통계적 오차 존재 (이벤트 수 의존)
  • 계산 시간: 수 초~수 분
  • 가장 정확한 방법 (Gold Standard)

해석적 계산 (Analytical)

  • 공식 기반 계산 (빌드업 팩터 근사)
  • 1D 근사 (넓은 빔 가정)
  • 결정론적 (통계 오차 없음)
  • 계산 시간: 즉시 (<1 ms)
  • 빠른 교차 검증 + 설계 도구 기반
검증 결과: MC/해석적 비율은 일반적으로 0.9~1.1 (±10%) 범위.
Co-60 납 5cm 예시: MC 31.2 mSv/h vs 해석적 29.8 mSv/h = +4.7% (PASS).
±20% 초과 시 경고 표시 — 기하 설정 또는 물리 모델 확인 필요.

10차폐 설계 도구

P2.12wc에서 구현된 LT-14 Inverse Shield Design Tool. "목표 선량률을 달성하려면 차폐가 얼마나 두꺼워야 하는가?"라는 역문제(inverse problem)를 풉니다.

역문제 개요

순문제 (Forward Problem) — 기존 방식
  입력: 핵종, 방사능, 차폐 물질, 두께, 거리
  출력: 선량률

역문제 (Inverse Problem) — 차폐 설계 도구
  입력: 핵종, 방사능, 차폐 물질, 목표 선량률, 거리
  출력: 필요 두께

  ────────────────────────────────────────────

  알고리즘: 이분법 (Bisection Method)
  ┌──────────────────────────────────────────┐
  │  초기 범위: [0 cm, 200 cm]              │
  │  반복 횟수: 최대 100회                   │
  │  수렴 조건: 상대오차 < 0.1%             │
  │                                          │
  │  while (|DoseRate(x) - target| > 0.1%): │
  │    mid = (low + high) / 2                │
  │    if DoseRate(mid) > target:            │
  │      low = mid     (더 두꺼워야 함)      │
  │    else:                                 │
  │      high = mid    (충분히 두꺼움)        │
  │                                          │
  │  → 수렴된 mid가 필요 두께                │
  └──────────────────────────────────────────┘

규제 기준 프리셋

프리셋연간 선량 한도시간당 선량률 (2000h 기준)적용 대상
Public 1 mSv/y 0.5 μSv/h 일반인 비관리구역

  ┌────────────────────┬─────────────────────┐

12빌드 시스템

빌드(Build)란 사람이 읽는 소스코드(C++)를 컴퓨터가 실행할 수 있는 바이너리로 변환하는 과정입니다.

소스코드              컴파일러           바이너리
main.cc          ─→  clang++       ─→  ShieldDose
src/*.cc              (Apple M5)        (실행 파일)
  +
GEANT4 라이브러리  ─→  링커          ─→  (GEANT4 코드 포함)

CMake의 역할

CMake는 이 변환 과정의 "레시피"를 정의합니다:

# CMakeLists.txt — "이 재료로 이 요리를 만들어라"

# 필요한 재료 (라이브러리) 찾기
find_package(Geant4 REQUIRED)     # GEANT4 라이브러리 위치 찾기

# 완성품 (실행 파일) 정의
add_executable(ShieldDose         # 이름: ShieldDose
    main.cc                       # 재료 1: 메인 코드
    src/DetectorConstruction.cc   # 재료 2: 기하 구성
    src/RunAction.cc              # 재료 3: 실행 동작
    ...                           # 나머지 소스 파일
)

# 라이브러리 연결 (링크)
target_link_libraries(ShieldDose
    ${Geant4_LIBRARIES}           # GEANT4와 결합
)

두 개의 독립된 빌드

CLI 빌드 (GEANT4 필요)

cd build/
cmake ..
make -j4
→ ShieldDose (CLI 바이너리)

GEANT4 + 소스코드 → 시뮬레이션 실행 파일

GUI 빌드 (Qt5만 필요)

cd ShieldDose_GUI_v3/build/
cmake ..
make -j4
→ ShieldDoseGUI (GUI 바이너리)

Qt5 + GUI 소스코드 → 사용자 인터페이스 실행 파일

왜 분리?
CLI 빌드에는 GEANT4 설치가 필수 (수 GB, 복잡한 의존성).
GUI 빌드에는 Qt5만 필요 (brew install qt@5, 간단).
분리하면 GUI 개발 시 GEANT4 빌드를 기다릴 필요 없음.

14왜 이렇게 설계하는가

핵심 질문: "GUI에 GEANT4를 직접 넣으면 안 되나?"

직접 임베드 (Phase 2 방식)

  • GUI + GEANT4가 같은 프로세스에서 실행
  • 시뮬레이션 중 GUI 멈춤 (이벤트 루프 충돌)
  • GEANT4 크래시 → GUI도 같이 죽음
  • GUI 코드 수정 → 물리 검증 재필요
  • 빌드 복잡 (Qt5 + GEANT4 동시 링크)
  • 결과: 25개 결함, 실질적 미작동

프로세스 분리 (Phase 3 방식)

  • GUI와 CLI가 별도 프로세스
  • 시뮬레이션 중 GUI 정상 반응 (비동기)
  • CLI 크래시 → GUI에서 에러 표시 (안전)
  • GUI 수정해도 물리 엔진 불변 (검증 유지)
  • 빌드 단순 (각각 독립 빌드)
  • 검증된 CLI 그대로 활용

실제 사례: 이 방식을 쓰는 다른 프로젝트들

프로젝트방식설명
TOPAS (MC 시뮬레이터) 매크로 파일 기반 GEANT4를 감싸서 간단한 텍스트 설정으로 사용. 세계적으로 가장 널리 쓰이는 의학물리 MC 도구.
GATE (의학영상 MC) 스크립트 기반 GEANT4 위에 매크로 레이어를 추가. PET/SPECT 시뮬레이션에 표준.
FLUKA 입력 카드 기반 CERN의 또 다른 MC 코드. 텍스트 입력 파일로 시뮬레이션 정의.
결론:
ShieldDose = CERN GEANT4의 모든 물리적 정확도를 유지하면서,
방사선 방호 전문가가 C++ 코딩 없이 4단계로 시뮬레이션할 수 있게 만드는 프로젝트.

GUI는 GEANT4의 "리모컨"이지, GEANT4의 "대체품"이 아닙니다.
